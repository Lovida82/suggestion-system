# 제안서 유형별 테스트 체크리스트

## 테스트 목적
제안 시스템의 4가지 유형별 업무 플로우를 검증하고, 각 권한별 사용자가 정상적으로 기능을 사용할 수 있는지 확인

## 테스트 환경
- **테스트 인원**
  - 일반 사용자 1명
  - 유형 평가자 1명 (유형효과 검증담당자)
  - 리더 1명 (소위원회 - 1차 평가자)
  - 리더 1명 (제안위원회 - 2차 평가자)

## 제안서 유형 분류

### 유형 1: 유형효과 없음 + 공동제안자 없음
**특징**: 가장 단순한 제안서 플로우

### 유형 2: 유형효과 없음 + 공동제안자 있음
**특징**: 공동제안자 관리 기능 검증

### 유형 3: 유형효과 있음 + 공동제안자 없음
**특징**: 유형효과 검증 프로세스 포함

### 유형 4: 유형효과 있음 + 공동제안자 있음
**특징**: 모든 기능 포함된 완전한 플로우

---

## 유형 1: 유형효과 없음 + 공동제안자 없음

### A. 일반 사용자 (제안자)

#### 1. 제안서 작성
- [ ] 로그인 → 대시보드 진입
- [ ] "새 제안서 작성" 버튼 클릭
- [ ] 기본 정보 입력
  - [ ] 제안 제목 입력
  - [ ] 카테고리 선택
  - [ ] 우선순위 선택
- [ ] 제안 내용 입력
  - [ ] 기존실시내용 입력
  - [ ] 변경(신규)내용 입력
- [ ] 실시 정보 입력
  - [ ] 실시자 선택
  - [ ] 실시부서 선택
- [ ] 유형효과 설정
  - [ ] "유형효과 없음" 선택 확인
- [ ] 공동제안자
  - [ ] 공동제안자 미추가 확인
- [ ] 자체 평가
  - [ ] 노력도 점수 입력
  - [ ] 창의성 점수 입력
  - [ ] 품질 점수 입력
  - [ ] 안전 점수 입력
  - [ ] 총점 및 등급 자동 계산 확인
- [ ] 제안서 제출
  - [ ] "제출" 버튼 클릭
  - [ ] 제출 확인 메시지 표시

#### 2. 제안서 조회 및 상태 확인
- [ ] "나의 제안" 메뉴에서 제출한 제안서 확인
- [ ] 상세보기 버튼 클릭하여 내용 확인
- [ ] 상태가 "검토중" 또는 "결재대기"로 표시되는지 확인
- [ ] 제안번호 자동 부여 확인

---

### B. 리더 사용자 - 소위원회 (1차 평가자)

#### 3. 제안서 검토 및 평가
- [ ] 로그인 → 리더 대시보드 진입
- [ ] "검토 대기" 메뉴 클릭
- [ ] 해당 제안서 확인
- [ ] "검토하기" 버튼 클릭
- [ ] 제안서 내용 확인
  - [ ] 제안 정보 표시 확인
  - [ ] 자체평가 점수 확인
- [ ] 평가 점수 입력
  - [ ] 노력도 평가
  - [ ] 창의성 평가
  - [ ] 품질 평가
  - [ ] 안전 평가
  - [ ] 총점 및 등급 자동 계산 확인
- [ ] 평가 의견 입력 (선택)
- [ ] 승인 처리
  - [ ] "승인" 버튼 클릭
  - [ ] 승인 완료 메시지 확인

#### 4. 전체 제안 목록 상세보기
- [ ] "전체 제안" 메뉴 클릭
- [ ] 제안서 목록 표시 확인
- [ ] "상세보기" 버튼 클릭
- [ ] 모달에서 제안서 정보 정상 표시 확인

---

### C. 리더 사용자 - 제안위원회 (2차 평가자)

#### 5. 최종 평가
- [ ] 로그인 → 리더 대시보드 진입
- [ ] "검토 대기" 메뉴 클릭
- [ ] 1차 평가 완료된 제안서 확인
- [ ] "검토하기" 버튼 클릭
- [ ] 1차 평가 결과 확인
- [ ] 최종 평가 점수 입력
- [ ] 최종 승인
  - [ ] "승인" 버튼 클릭
  - [ ] 최종 승인 완료 메시지 확인

#### 6. 승인 완료 확인
- [ ] "검토 완료" 메뉴에서 승인된 제안서 확인
- [ ] 최종 상태 "승인" 확인

---

## 유형 2: 유형효과 없음 + 공동제안자 있음

### A. 일반 사용자 (제안자)

#### 1. 제안서 작성 (공동제안자 포함)
- [ ] 로그인 → 대시보드 진입
- [ ] "새 제안서 작성" 버튼 클릭
- [ ] 기본 정보 입력 (유형 1과 동일)
- [ ] **공동제안자 추가**
  - [ ] "공동제안자 추가" 버튼 클릭
  - [ ] 공동제안자 1 정보 입력
    - [ ] 이름, 부서, 역할 입력
    - [ ] 기여도(%) 입력
  - [ ] 공동제안자 2 정보 입력 (선택)
    - [ ] 이름, 부서, 역할 입력
    - [ ] 기여도(%) 입력
  - [ ] **기여도 합계 100% 검증**
    - [ ] 100% 미만/초과 시 경고 메시지 확인
    - [ ] 정확히 100%일 때만 제출 가능 확인
- [ ] 제안서 제출
- [ ] 공동제안자 정보가 저장되었는지 확인

#### 2. 공동제안자 정보 확인
- [ ] "나의 제안" 메뉴에서 제출한 제안서 확인
- [ ] 상세보기에서 공동제안자 목록 표시 확인
- [ ] 각 공동제안자의 이름, 역할, 기여도 표시 확인

---

### B. 리더 사용자 - 소위원회 & 제안위원회

#### 3. 공동제안자 포함 제안서 검토
- [ ] 제안서 검토 시 공동제안자 정보 확인
- [ ] 공동제안자 테이블 정상 표시 확인
- [ ] 평가 및 승인 프로세스 (유형 1과 동일)

---

## 유형 3: 유형효과 있음 + 공동제안자 없음

### A. 일반 사용자 (제안자)

#### 1. 유형효과 포함 제안서 작성
- [ ] 로그인 → 대시보드 진입
- [ ] "새 제안서 작성" 버튼 클릭
- [ ] 기본 정보 입력
- [ ] **유형효과 설정**
  - [ ] "유형효과 있음" 체크
  - [ ] 예상 절감액 입력 (원/년)
  - [ ] 산출 근거 입력
- [ ] 제안서 제출
- [ ] 제출 완료 메시지 확인

#### 2. 유형효과 검증 대기 상태 확인
- [ ] "나의 제안" 메뉴에서 제안서 확인
- [ ] 상태가 "유형효과 검증 대기" 또는 유사 상태 확인
- [ ] 상세보기에서 유형효과 정보 확인

---

### B. 유형 평가자 (유형효과 검증담당자)

#### 3. 유형효과 검증
- [ ] 로그인 → 유형효과 검증 시스템 진입
- [ ] "검증 대기" 메뉴 클릭
- [ ] 해당 제안서 확인
- [ ] "검증하기" 버튼 클릭
- [ ] 제안서 정보 확인
  - [ ] 제안 내용 확인
  - [ ] 예상 절감액 및 산출 근거 확인
- [ ] **AI 기준 요청 (선택)**
  - [ ] "AI 기준 요청하기" 버튼 클릭
  - [ ] AI 분석 결과 표시 확인
  - [ ] 분석 내용 검토
- [ ] 확정 금액 입력
  - [ ] 확정 금액(원/년) 입력
  - [ ] 자동 계산된 유형효과 점수 확인
- [ ] 금액 조정 사유 입력 (예상 금액과 다를 경우)
- [ ] 검증 의견 입력 (선택)
- [ ] **검증 완료**
  - [ ] "검증 완료" 버튼 클릭
  - [ ] 검증 완료 메시지 확인

#### 4. 검증 완료 확인
- [ ] "검증 완료" 메뉴에서 처리한 제안서 확인
- [ ] 확정 금액 및 조정률 확인

#### 5. 전체 제안 목록 상세보기
- [ ] "전체 제안" 메뉴 클릭
- [ ] 제안서 목록 표시 확인
- [ ] **"상세보기" 버튼 클릭** ← **수정된 기능**
- [ ] 모달에서 제안서 정보 정상 표시 확인
- [ ] 유형효과 정보 표시 확인

---

### C. 리더 사용자 - 소위원회 (1차 평가자)

#### 6. 유형효과 검증 완료된 제안서 검토
- [ ] "검토 대기" 메뉴에서 제안서 확인
- [ ] 제안서 검토 시 유형효과 검증 결과 확인
  - [ ] 확정 금액 표시 확인
  - [ ] 유형효과 점수 확인
- [ ] 평가 점수 입력 (유형 1과 동일)
- [ ] 승인 처리

---

### D. 리더 사용자 - 제안위원회 (2차 평가자)

#### 7. 최종 평가 (유형 1과 동일)
- [ ] 최종 평가 및 승인
